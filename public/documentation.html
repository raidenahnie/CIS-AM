Introduction

CIS-AM (Curriculum Implementation System - Attendance Monitoring) is a specialized web-based attendance management system developed for the Department of Education - Cavite Province, specifically designed for use by the Curriculum Implementation Division.

This system provides accurate, GPS-verified attendance tracking for DepEd Cavite personnel, ensuring accountability and efficient workforce management across multiple school locations and division offices throughout Cavite Province.

Employee Usage Guide

User Account Registration

To register an account, please contact your system administrator to register you in the system.

1. Login

  1. Navigate to the login page
  2. Enter your email and password
  3. Click "Sign In"

2. Check In

  1. Select your workplace from the dashboard
  2. Click "Check In" button
  3. Allow location access when prompted
  4. System verifies you're within the workplace radius
Manual Entry Option

If GPS is unavailable, you can use the Manual Entry option with the access code provided by your administrator.

3. Break Management

  1. Click "Start Break" when taking a break
  2. Click "End Break" when returning to work
  3. Break duration is automatically calculated

4. Check Out

  1. Click "Check Out" when ending your work day
  2. System calculates total hours worked (minus break time)

5. View Attendance History

  1. Navigate to "Attendance History" section
  2. View detailed logs of all activities
  3. Filter by date range

Administrator Guide

1. Dashboard Overview

  • View real-time statistics (check-ins, late arrivals, etc.)
  • Monitor current employee status
  • Quick access to all management functions

2. User Management

  • Add new employees with role assignment
  • Edit user information and permissions
  • Deactivate/activate user accounts
  • Perform bulk operations for efficiency

3. Workplace Management

  • Create new workplace locations
  • Set GPS coordinates and check-in radius
  • Assign users to workplaces
  • View workplace statistics

4. Reports Generation

  1. Navigate to Reports section
  2. Select report type (Weekly/Monthly/Custom)
  3. Apply filters (employee, workplace, date range)
  4. Click "Generate Report"
  5. Export to CSV/Excel format

5. System Settings

  • Update manual entry access code
  • Configure admin account settings
  • Manage system parameters
Security Notice

All sensitive operations require admin password verification for security purposes.

Developer Guide

Overview

This section helps developers set up, run, and extend the CIS-AM application. It was generated from the current source files (controllers, models, routes and console commands) ‚Äî do not rely on the older docs or markdown files found elsewhere in the repository.

Local setup (Windows / XAMPP)

  1. Install PHP 8.2+, Composer, Node.js (18+), and XAMPP (Apache + MySQL).
  2. Copy .env.example to .env and update database and mail settings.
  3. Install PHP dependencies: composer install.
  4. Install JS dependencies and build assets: npm install then npm run dev (or npm run build for production).
  5. Create database and run migrations: php artisan migrate and (optionally) php artisan db:seed.
  6. Create session table if using DB sessions: php artisan session:table then php artisan migrate.
  7. Run the local server: php artisan serve or use XAMPP virtual host pointing to the project public/ folder.

Important environment variables

  • APP_URL, APP_ENV, APP_DEBUG
  • DB_CONNECTION, DB_HOST, DB_DATABASE, DB_USERNAME, DB_PASSWORD
  • MAIL_MAILER, MAIL_HOST, MAIL_PORT, MAIL_USERNAME, MAIL_PASSWORD
  • SESSION_DRIVER=database recommended for single-device enforcement
  • SMS integration: SMS_API_URL, SMS_API_KEY, SMS_TEST_PHONE
  • Manual entry / admin keys: stored via the SystemSetting model (see app/Models/SystemSetting.php).

Database

Key models (see app/Models):

  • User ‚Äî accounts and roles
  • Workplace, UserWorkplace ‚Äî locations and assignments
  • Attendance, AttendanceLog ‚Äî per-day summaries and per-action logs
  • AbsenceRequest ‚Äî user leave requests
  • AdminActivityLog ‚Äî admin auditing
  • PasswordReset, SystemSetting ‚Äî auxiliary systems

Console commands

Two custom Artisan commands are included in app/Console/Commands:

  • php artisan check:attendance-data ‚Äî prints today's attendance logs for debugging.
  • php artisan setup:pm-user {userId} ‚Äî sets up a PM test check-in for the specified user (default userId=1).

Run tests and quality checks

  • PHPUnit tests: vendor/bin/phpunit or php artisan test (project includes phpunit.xml).
  • Static analysis / linters: add phpstan / phpcs as desired.

Key classes & extension points

  • app/Http/Controllers/Api/DashboardController.php ‚Äî most employee-facing APIs and check-in logic live here (distance calculation, special vs regular flows, summary update).
  • app/Http/Controllers/AdminController.php ‚Äî admin CRUD, settings, bulk operations and notification settings.
  • app/Models/AttendanceLog.php and Attendance.php ‚Äî core domain models; most reporting logic reads these.
  • app/Http/Controllers/AdminReportController.php ‚Äî reports, exports and helpers to expand attendance rows for export.

Common developer tasks

  • Rebuild assets: npm run dev or npm run build.
  • Run migrations: php artisan migrate.
  • Seed demo data: check database/seeders and run php artisan db:seed.
  • Start the local server: php artisan serve.
  • Clear caches: php artisan config:clear, php artisan cache:clear, php artisan route:clear.

Security & production notes

  • Use secure values for APP_KEY, MAIL and SMS keys in production.
  • Session driver should be database for single-device login enforcement used by the app.
  • Ensure cron/scheduler runs for scheduled tasks if any: php artisan schedule:run (set up in server cron).
